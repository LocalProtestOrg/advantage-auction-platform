## What It Does

Renders a responsive card grid of featured auction lots pulled from the Advantage platform.
Cards include lot photo, auction badge (LIVE NOW / UPCOMING / ENDING SOON), lot title,
location, current bid with buyer premium, and a direct link to the lot detail page.

A "Sell with Advantage" CTA card appears after the last lot card by default. The CTA
can be disabled via config.

The grid renders a skeleton loading state immediately on page load, then populates with
live data. If no featured lots are available, an empty state message is shown instead of
a broken container.

| Partner landing page | Main body, after an introductory paragraph |
| Auction-category page | After category description, before navigation links |
| Email landing pages | Top of page — this is the first thing buyers should see |

Ideal container width: **1000px–1400px**. The grid auto-fills columns at ~280px minimum
card width. At 1100px wide you get approximately 3 columns; at 900px, 2-3 columns.

Do not place inside a narrow sidebar or a column narrower than 320px — cards will stack
to a single column and may appear too tall relative to the page.

## Embed Code: Standalone (Recommended for Single-Widget Pages)

Copy this exactly. Paste the `<div>` where you want the widget to appear.
Paste the `<script>` tag immediately after the closing `</div>` of the widget container.

```html
<!-- Advantage Featured Lots Widget — standalone embed -->
<div
  id="aap-featured-lots"
  data-api-base="https://auctions.advantage.bid"
  data-limit="6"
  data-auction-state="published"
></div>
<script src="https://auctions.advantage.bid/widgets/featured-lots.js" defer></script>
```

**No other scripts are needed.** The standalone embed bundles all required shared code.

## `data-*` Configuration Attributes

All attributes are set on the `<div id="aap-featured-lots">` container element.

| Attribute | Type | Default | Description |
|---|---|---|---|
| `data-api-base` | string | `""` | Base URL for API calls. Always set to `https://auctions.advantage.bid` on BD pages. |
| `data-limit` | string (integer) | `"6"` | Number of lot cards to display. Must be a quoted integer: `"6"`, not `6`. |
| `data-auction-state` | string | `"published"` | Filter by auction state: `"published"`, `"active"`, or `"closed"`. |
| `data-theme` | string | `""` | Set to `"dark"` to enable dark card theme. |
| `data-show-cta` | string | `"true"` | Set to `"false"` to hide the "Sell with Advantage" CTA card. |

### Notes on `data-limit`

The value **must be a string containing an integer**: `data-limit="6"`. If you pass
a non-integer string or omit the attribute, the widget falls back to the AAPConfig value,
then to the platform default of 6.

## AAPConfig Keys

Config keys that apply to this widget when using the full platform layer.

| Key | Type | Default | Description |
|---|---|---|---|
| `widget.featured-lots.limit` | number | `6` | Default card limit (lower priority than data-limit) |
| `marketplace.show_cta` | boolean | `true` | Show/hide the seller CTA card |
| `marketplace.cta_url` | string | `""` | Destination URL for the CTA card button |
| `marketplace.cta_headline` | string | `""` | CTA card headline (falls back to platform default) |
| `marketplace.cta_body` | string | `""` | CTA card body text |
| `marketplace.cta_button_label` | string | `"Get Started"` | CTA button label |
| `marketplace.currency` | string | `"USD"` | Currency code for bid display |

**Priority:** `data-*` attribute > `AAPConfig.set()` > remote config > platform default.

## Required API Endpoints

| Endpoint | Method | Auth | Used for |
|---|---|---|---|
| `GET /api/public/featured-lots` | GET | None | Fetches featured lot cards |
| `GET /api/public/config` | GET | None | Optional remote config loading |

### Featured Lots Response Shape

```json
{
  "success": true,
  "data": [
    {
      "id": "uuid",
      "title": "Lot title",
      "image_url": "https://...",
      "current_bid": 150,
      "buyer_premium_pct": 15,
      "auction_id": "uuid",
      "auction_title": "Auction name",
      "auction_state": "published",
      "ends_at": "2026-06-01T18:00:00Z",
      "location_city": "Dallas",
      "location_state": "TX",
      "lot_url": "https://auctions.advantage.bid/auctions/slug/lots/slug"
    }
  ]
}
```

The widget uses `auction_state` and `ends_at` to compute badge type (LIVE NOW / UPCOMING /
ENDING SOON). It uses `current_bid` and `buyer_premium_pct` to display the all-in price.

3. Paste the widget embed code in the same position.

## Mobile Considerations

| Breakpoint | Behavior |
|---|---|
| 480px and above | CSS Grid auto-fill at `minmax(280px, 1fr)` |
| Below 480px | Single-column grid |

The card image aspect ratio is fixed at 16:9 and will not distort at any width.
Cards are touch-friendly — no hover-only interactions for core navigation.

**Test at 375px width** (iPhone SE baseline) before marking a deployment complete.
The single-column layout should fill the viewport without horizontal scroll.

## Accessibility

- All lot cards are `<a>` elements with meaningful `href` — keyboard-navigable by default.
- Card images include `alt` text from the lot title.
- LIVE NOW / UPCOMING badges include `aria-label` with human-readable status.
- The skeleton loading state uses `aria-busy="true"` on the grid container.
- Color contrast for text on card backgrounds meets WCAG AA.
- The CTA card button is a standard `<button>` element with descriptive label.

No additional accessibility configuration is needed on the BD page.

## SEO Considerations

- The widget renders content client-side via JavaScript. Server-side search engine crawlers
  may not index the lot cards.
- For SEO-critical pages, add a brief static introduction paragraph above the widget
  describing the page content (e.g., "Browse featured auction lots from Advantage.Bid").
- The widget's lot links (`lot_url`) are standard anchor tags and will be followed by
  crawlers that do execute JavaScript.
- Do not rely on the widget grid content for your primary page SEO signals — treat it
  as dynamic commerce content, not editorial content.

---

## Cache Considerations

- The widget fetches fresh data from the API on every page load.
- The API response for `/api/public/featured-lots` is served with `Cache-Control: s-maxage=60`.
  CDN/edge caches may serve the response for up to 60 seconds before revalidating.
- Browser cache on the widget script itself is `s-maxage=3600` (1 hour).
  Script updates roll out within 1 hour of engineering deploying a new version.
- If a BD page uses `AAPConfig.loadRemote()`, the remote config response is cached
  in localStorage for 300 seconds (5 minutes) by default. The widget will use the
  cached config until the TTL expires or the cache is invalidated.
- Hard refresh (`Ctrl+Shift+R`) bypasses browser cache but not CDN cache.

## Rollback Instructions

### Quick rollback (config issue — wrong data-* attribute)

1. Correct the `data-*` attribute on the container `<div>`.
2. No script changes needed.
3. Hard refresh and verify.

### Full rollback (widget failure, layout break, or JS error)

1. Open `deployment-log.md` and locate the rollback snapshot for this deployment.
2. Remove the widget `<div>` and its `<script>` tag from the BD page.
   - If using the full platform layer and no other widgets remain: also remove all shared `<script>` tags.
3. Paste the saved rollback snapshot HTML back in place.
4. Hard refresh and verify the page looks correct.
5. Add a rollback entry to `deployment-log.md`:

```markdown
## [Date] — ROLLBACK: widget-featured-lots v1.0.0 on [BD Page]
- Reason: [brief description]
- Restored to: pre-deployment state
- Engineering notified: [yes/no]
```

6. Notify engineering if the cause was a widget bug (not a config error).

## Known Limitations

- The grid does not support a `data-columns` attribute — column count is auto-calculated
  based on available container width. To force a column count, wrap the container in a
  fixed-width parent.
- Version-pinned CDN URLs (e.g., `/widgets/v1.0.0/featured-lots.js`) are not available.
  The script at `/widgets/featured-lots.js` always serves the latest stable version.
- The widget cannot be rendered server-side — it requires JavaScript to be enabled.

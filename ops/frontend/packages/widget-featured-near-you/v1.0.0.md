## What It Does

Renders a responsive card grid of featured auctions filtered by proximity to the buyer's
current location. The buyer's location is resolved in this priority order:

1. **`data-lat` + `data-lng` attributes** — hardcoded coordinates on the container (e.g., for a regional landing page)
2. **Browser Geolocation API** — requested at page load if coordinates are not hardcoded
3. **IP geolocation fallback** — used if the browser denies the geolocation request, or if the API call times out

If none of the three sources produce a location, the widget renders without a location
filter and shows featured auctions ordered by relevance.

Cards are identical to the Featured Lots widget: lot photo, badge, title, location,
bid price, and a direct link. A seller CTA card appears after the lot cards by default.

## Embed Code: Standalone — Geo Auto-Detect (Recommended)

The widget requests the browser's location on load. If the user grants permission,
the API is called with their coordinates. If denied, IP fallback is used.

```html
<!-- Advantage Featured Near You Widget — standalone, auto-detect location -->
<div
  id="aap-featured-near-you"
  data-api-base="https://auctions.advantage.bid"
  data-limit="6"
  data-radius-km="200"
></div>
<script src="https://auctions.advantage.bid/widgets/featured-near-you.js" defer></script>
```

## `data-*` Configuration Attributes

All attributes are set on the `<div id="aap-featured-near-you">` container element.

| Attribute | Type | Default | Description |
|---|---|---|---|
| `data-api-base` | string | `""` | Base URL for API calls. Always set to `https://auctions.advantage.bid` on BD pages. |
| `data-limit` | string (integer) | `"6"` | Number of cards to display. Must be a quoted integer: `"6"`. |
| `data-radius-km` | string (integer) | `"200"` | Search radius in kilometers. Must be a quoted integer. |
| `data-lat` | string (float) | `""` | Latitude to center geo search. If set, skips browser geolocation. |
| `data-lng` | string (float) | `""` | Longitude to center geo search. Required if `data-lat` is set. |
| `data-theme` | string | `""` | Set to `"dark"` for dark card theme. |
| `data-show-cta` | string | `"true"` | Set to `"false"` to hide the "Sell with Advantage" CTA card. |

### Coordinate Requirements

- `data-lat` and `data-lng` must both be present, or neither. Setting one without the other
  falls back to browser geolocation.
- Values must be decimal degree strings: `"32.7767"`, not `"32° 46' 36\""`.
- Valid range: latitude `-90` to `90`, longitude `-180` to `180`.

## Location Resolution Strategy (Detail)

The widget resolves location using three strategies, tried in order:

### Strategy 1: Hardcoded coordinates
If `data-lat` and `data-lng` are both set on the container, the widget uses them directly
and makes no geolocation request. The browser permission dialog is never shown.

### Strategy 2: Browser Geolocation API
If no coordinates are hardcoded, the widget calls `navigator.geolocation.getCurrentPosition()`.
The browser will show its standard location permission dialog.

- If granted: coordinates are sent to the API as query parameters.
- If denied or the API is unavailable: falls through to Strategy 3.
- Timeout: 5 seconds. If geolocation takes longer, falls through to Strategy 3.

### Strategy 3: IP geolocation fallback
The widget calls the API without coordinates. The server resolves approximate location
from the request's IP address. Results are less precise (city-level or broader),
and location accuracy degrades for users on VPNs or shared IP ranges.

If none of the three strategies produces usable data, the widget renders all featured
auctions without geo filtering, ordered by relevance.

## Required API Endpoints

| Endpoint | Method | Auth | Used for |
|---|---|---|---|
| `GET /api/public/featured-near-you` | GET | None | Fetches geo-filtered auctions |
| `GET /api/public/config` | GET | None | Optional remote config loading |

### Query Parameters for `/api/public/featured-near-you`

| Parameter | Type | Description |
|---|---|---|
| `lat` | float | Latitude of search center (optional) |
| `lng` | float | Longitude of search center (optional) |
| `radius_km` | integer | Search radius in kilometers (optional, default 200) |
| `limit` | integer | Max results to return (optional, default 6) |
| `state` | string | Filter by auction state (optional, default "published") |

### Response Shape

```json
{
  "success": true,
  "data": [
    {
      "id": "uuid",
      "title": "Auction title",
      "image_url": "https://...",
      "current_bid": 75,
      "buyer_premium_pct": 15,
      "auction_id": "uuid",
      "auction_title": "Auction name",
      "auction_state": "published",
      "ends_at": "2026-06-15T14:00:00Z",
      "location_city": "Dallas",
      "location_state": "TX",
      "distance_km": 12.4,
      "lot_url": "https://auctions.advantage.bid/auctions/slug/lots/slug"
    }
  ]
}
```

`distance_km` is included when coordinates were provided. Cards display the distance
as "12 km away" when available.

document.addEventListener('DOMContentLoaded', function () {

## Mobile Considerations

| Breakpoint | Behavior |
|---|---|
| 480px and above | CSS Grid auto-fill at `minmax(280px, 1fr)` |
| Below 480px | Single-column grid |

The geolocation permission dialog is triggered by the browser — it cannot be styled or
suppressed. On mobile, this dialog appears at the top of the screen and does not block
the page. The widget renders the skeleton loading state while waiting for permission.

If the user denies location permission on mobile, the IP fallback activates without any
visible error to the user.

**Test at 375px width** before marking a deployment complete.

## Accessibility

- All lot cards are `<a>` elements — keyboard-navigable.
- Card images include `alt` text from the lot title.
- Distance text ("12 km away") is visible text, not an icon-only indicator.
- Badges include `aria-label` attributes.
- The skeleton state uses `aria-busy="true"` on the grid container.
- The geolocation permission dialog is a native browser dialog — fully accessible by default.

---

## SEO Considerations

- Widget content is rendered client-side. Search engines may not index the card grid.
- For regional SEO pages, the surrounding editorial content (headlines, paragraphs) carries
  the SEO weight. The widget provides commerce value, not keyword density.
- If `data-lat` / `data-lng` are hardcoded, consider adding a visible regional headline
  above the widget: "Featured Auctions Near Dallas, TX" — this phrase carries SEO value
  even when the grid itself is not indexed.
- Lot links within the grid are standard anchor tags and will be followed by JS-capable crawlers.

---

## Cache Considerations

- Widget fetches fresh data on every page load.
- `/api/public/featured-near-you` API responses are NOT cached at the CDN layer — location
  is dynamic (different coordinates per request). Response is served fresh from the origin.
- The widget script itself (`featured-near-you.js`) is cached at the CDN for 1 hour.
- Browser geolocation permission is cached by the browser — once granted for a domain,
  it remains granted until the user revokes it in browser settings.

## Rollback Instructions

### Quick rollback (wrong coordinates or radius)

1. Correct the `data-lat`, `data-lng`, or `data-radius-km` attribute on the container.
2. Hard refresh and verify.

### Disabling geolocation prompt (user experience issue)

Add hardcoded regional coordinates to bypass browser geolocation entirely:
```html
data-lat="32.7767" data-lng="-96.7970"
```

This prevents the browser permission dialog from appearing.

### Full rollback (widget failure, layout break, or JS error)

1. Open `deployment-log.md` and locate the rollback snapshot for this deployment.
2. Remove the widget `<div>` and its `<script>` tag from the BD page.
   - If using full platform layer and no other widgets remain: remove all shared `<script>` tags.
3. Paste the saved rollback snapshot HTML back in place.
4. Hard refresh and verify the page looks correct.
5. Add a rollback entry to `deployment-log.md`:

```markdown
## [Date] — ROLLBACK: widget-featured-near-you v1.0.0 on [BD Page]
- Reason: [brief description]
- Restored to: pre-deployment state
- Engineering notified: [yes/no]
```

6. Notify engineering if the cause was a widget bug (not a config or coordinate error).

## Known Limitations

- Geolocation precision degrades for users on VPNs — IP fallback will place them at
  their VPN exit node, which may be hundreds of miles from their actual location.
- The widget cannot force a geolocation prompt to re-appear once the user has denied it.
  The IP fallback will be used silently for those users.
- No `data-columns` attribute — columns are auto-calculated by CSS Grid.
- Version-pinned CDN URLs are not available. `/widgets/featured-near-you.js` always
  serves the latest stable version.

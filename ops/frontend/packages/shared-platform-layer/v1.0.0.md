## What This Package Contains

The shared platform layer is not a visible widget — it is the infrastructure foundation
that all Advantage widgets depend on. Load it once per page, before any widget scripts.

| Script | Purpose |
|---|---|
| `shared/utils.js` | DOM helpers, fetch wrapper, date/currency formatting, event bus |
| `shared/config.js` | AAPConfig v2 — key/value store with remote loading, local override preservation, localStorage cache |
| `shared/components/badge.js` | LIVE NOW, UPCOMING, ENDING SOON, SHIPS badges + injects root CSS variables |
| `shared/components/skeleton-card.js` | Animated loading skeleton card |
| `shared/components/auction-card.js` | Full auction/lot card with image, title, badges, pricing, location |
| `shared/components/seller-cta.js` | "Sell with Advantage" CTA card (appears at end of grid) |
| `shared/components/empty-state.js` | Empty state (no results) and error state rendering |

---

## When to Use This Package

Use the shared platform layer when:
- You are deploying **two or more widgets on the same page**
- You want to configure all widgets once using `AAPConfig.set()` instead of per-widget `data-*` attributes
- You want to load shared dependencies once and reduce HTTP requests

For single-widget deployments, the standalone embed in each widget package already includes
all required shared code bundled together. The standalone embed is self-sufficient.

## Script Load Order (Required)

The load order is strict. Do not rearrange.

```html
<!-- 1. Utilities (no dependencies) -->
<script src="https://auctions.advantage.bid/widgets/shared/utils.js"></script>

<!-- 2. Config system (depends on utils) -->
<script src="https://auctions.advantage.bid/widgets/shared/config.js"></script>

<!-- 3. Configure before widgets run (optional but must come before component scripts if used) -->
<script>
  AAPConfig.set({
    'marketplace.currency': 'USD',
    'widget.featured-lots.limit': 6,
    'widget.featured-near-you.limit': 6
  });
</script>

<!-- 4. Component library (depends on utils + config) -->
<script src="https://auctions.advantage.bid/widgets/shared/components/badge.js"></script>
<script src="https://auctions.advantage.bid/widgets/shared/components/skeleton-card.js"></script>
<script src="https://auctions.advantage.bid/widgets/shared/components/auction-card.js"></script>
<script src="https://auctions.advantage.bid/widgets/shared/components/seller-cta.js"></script>
<script src="https://auctions.advantage.bid/widgets/shared/components/empty-state.js"></script>

<!-- 5. Widget scripts (depend on all of the above) -->
<script src="https://auctions.advantage.bid/widgets/featured-lots.js"></script>
<script src="https://auctions.advantage.bid/widgets/featured-near-you.js"></script>
```

## AAPConfig Reference

`AAPConfig` is the global configuration object injected by `shared/config.js`. It persists
for the lifetime of the page and is shared across all widget instances.

### Core Methods

| Method | Description |
|---|---|
| `AAPConfig.set(key, value)` | Set a single config key |
| `AAPConfig.set({ key: value, ... })` | Set multiple config keys at once |
| `AAPConfig.get(key)` | Get a config value (returns `null` if not set) |
| `AAPConfig.get(key, fallback)` | Get a config value with a fallback default |
| `AAPConfig.loadRemote(url, opts)` | Load config from a remote JSON API; returns a Promise |
| `AAPConfig.invalidateCache()` | Clear the localStorage config cache |
| `AAPConfig.dumpOverrides()` | Returns a copy of all locally-set key/value overrides |

### `loadRemote()` Options

```javascript
AAPConfig.loadRemote('https://auctions.advantage.bid/api/public/config', {
  cacheTtlSeconds: 300,   // default: 300 — how long to trust a cached response
  bypassCache: false      // default: false — set true to force a fresh fetch
});
```

The `loadRemote()` call always resolves (never rejects). If the fetch fails, the page
continues using defaults and any locally-set values.

### Remote Config Namespace Guard

`loadRemote()` only merges keys with these prefixes:

| Prefix | Example |
|---|---|
| `marketplace.*` | `marketplace.currency`, `marketplace.show_cta` |
| `widget.*` | `widget.featured-lots.limit`, `widget.featured-near-you.radius_km` |
| `analytics.*` | `analytics.gtm_id` |

Arbitrary keys from the remote endpoint are silently dropped.

### Local Override Priority

Keys set via `AAPConfig.set()` are never overwritten by `loadRemote()`.
If a BD page sets a key explicitly, that value is locked for the page session.

### Available Config Keys

| Key | Type | Default | Notes |
|---|---|---|---|
| `marketplace.currency` | string | `"USD"` | |
| `marketplace.show_cta` | boolean | `true` | Show "Sell with Advantage" CTA card |
| `marketplace.cta_url` | string | `""` | CTA card link target |
| `marketplace.cta_headline` | string | `""` | CTA card headline text |
| `marketplace.cta_body` | string | `""` | CTA card body text |
| `marketplace.cta_button_label` | string | `"Get Started"` | CTA card button label |
| `widget.featured-lots.limit` | number | `6` | Default lot limit (overridden by data-limit) |
| `widget.featured-near-you.limit` | number | `6` | Default lot limit |
| `widget.featured-near-you.radius_km` | number | `200` | Geo search radius |

## CSS Architecture

The shared platform layer injects all required CSS when components first run.
Injection is idempotent — each block has a unique `id` and will not be injected twice.

| Component | CSS injected | Style block ID |
|---|---|---|
| `badge.js` | Root CSS variables + badge styles | `aapc-root-styles`, `aapc-badge-styles` |
| `skeleton-card.js` | Pulse animation + skeleton card | `aapc-skel-styles` |
| `auction-card.js` | Card shell, body, typography | `aapc-card-styles` |
| `seller-cta.js` | CTA card styles | `aapc-cta-styles` |
| `empty-state.js` | Empty and error state styles | `aapc-state-styles` |

### CSS Namespace

All shared component CSS uses the `aapc-` prefix. Do not use this prefix in BD page CSS.
See `shared-css-strategy.md` for the full compatibility reference.

---

## No-Conflict Guarantee

- Scripts are idempotent — loading any script twice has no effect
- CSS blocks are idempotent — injected once and never duplicated
- `AAPConfig` is a singleton — multiple script loads share the same instance
- No global variables other than `AAPConfig` and `AAPWidgetUtils` are introduced

## Rollback

The shared platform layer has no BD-facing HTML. Rolling back means removing all
`<script>` tags for `shared/utils.js`, `shared/config.js`, and all component scripts —
**but only if no widget containers remain on the page.** If any widget `<div>` is present,
the widget script will fail without the platform layer.

Safe rollback sequence:
1. Remove all widget `<div>` containers and their `<script>` tags first
2. Then remove all platform layer `<script>` tags

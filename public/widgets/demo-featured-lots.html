Featured Lots Widget — Demo

Image-first lot showcase. Configurable via AAPConfig — zero hardcoded business variables.

Live demo — with shared platform layer

All business-facing variables (badge labels, CTA copy, display limits) come from AAPConfig. Data attributes on the container element override config values per-embed. A seller CTA card appears at the end of the grid when marketplace.cta.url is configured.

Data note: GET /api/public/featured-lots only returns lots marked is_featured = true on active or published auctions. If this demo shows empty state, ask an admin to mark some lots as featured via the admin interface.

Simple embed — single script, self-contained

Widget carries inline fallbacks for all components. Works with one script tag and no dependencies. Business variables fall back to platform defaults when AAPConfig is not loaded.

<div id="aap-featured-lots" data-api-base="https://auctions.advantage.bid" data-limit="6"> </div> <script src="https://auctions.advantage.bid/widgets/featured-lots.js"></script>

Full platform layer — recommended for multi-widget pages

Load shared utilities, config, and components once. All widgets on the page share one copy of each dependency. Configure marketplace variables centrally via AAPConfig.set().

Inline config block — alternative to AAPConfig.set()

Place a <script type="application/json" id="aap-config"> block anywhere on the page before widgets load. config.js reads it automatically.

<script id="aap-config" type="application/json"> { "marketplace.cta.url": "https://auctions.advantage.bid/seller-create.html", "marketplace.cta.headline": "Consigning an Estate?", "marketplace.badge.ending_soon_threshold_min": 90, "widget.limit": 8 } </script> <script src="https://auctions.advantage.bid/widgets/shared/config.js"></script> <!-- config.js reads the block above during its own initialization -->

Multi-widget page — featured lots + featured near you

Both widgets coexist on one page. Config and components are shared. CSS namespacing (aapfl- vs aapny-) prevents style collisions.

<!-- shared layer loaded once --> <script src=".../shared/utils.js"></script> <script src=".../shared/config.js"></script> <script>AAPConfig.set({ 'marketplace.cta.url': '...', 'widget.limit': 6 });</script> <script src=".../shared/components/badge.js"></script> <script src=".../shared/components/skeleton-card.js"></script> <script src=".../shared/components/auction-card.js"></script> <script src=".../shared/components/seller-cta.js"></script> <script src=".../shared/components/empty-state.js"></script> <script src=".../shared/components/error-state.js"></script> <!-- Widget A: Featured Near You --> <div id="aap-featured-near-you" data-api-base="..." data-use-geolocation="true"></div> <script src=".../widgets/featured-near-you.js"></script> <!-- Widget B: Featured Lots --> <div id="aap-featured-lots" data-api-base="..."></div> <script src=".../widgets/featured-lots.js"></script>

Analytics integration

var container = document.getElementById('aap-featured-lots'); // Fires after widget renders container.addEventListener('aap:widget:loaded', function (e) { // { widgetId: 'aap-featured-lots', resultCount: 6, source: 'featured-lots' } gtag('event', 'widget_loaded', e.detail); }); // Fires when a visitor clicks a lot card container.addEventListener('aap:lot:click', function (e) { // { lotId, lotTitle, auctionId, auctionTitle } gtag('event', 'lot_card_click', e.detail); // Route to your lot detail page using auctionId + lotId }); // Fires when CTA button is clicked container.addEventListener('aap:cta:click', function (e) { gtag('event', 'seller_cta_click', e.detail); });
